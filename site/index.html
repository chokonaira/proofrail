Hosted sandbox

Signed permits before agents act.

PermitRail sits between an AI agent and sensitive tools. It blocks risky actions until a purpose-bound proof exists, then writes a signed receipt for the audit trail.

Runnable today

A hostable UI for the real TypeScript demo.

The sandbox shows an agent attempting a suspicious payment from an untrusted email, then a legitimate email send. The payment is denied. The email requires approval, receives a signed proof, executes, and produces a signed receipt. 

npm install
npm run check
npm run demo
  • Proof is bound to subject, audience, purpose, and input hash.
  • Replay against changed tool input is denied by policy.
  • Every allow, block, or denial can produce a signed receipt.

Language-agnostic core

TypeScript SDK, portable protocol.

TypeScript-first

Strict types for policies, actions, proof envelopes, providers, and gateway results.

Signed JSON

Ed25519 envelopes make proofs and receipts verifiable outside the SDK.

Sidecar-ready

Java, Go, Python, and .NET services can call a PermitRail gateway over HTTP.

Approval roadmap

One permission model, multiple approval channels.

Local approval Passkey Email magic link Slack or Teams GitHub review Credential wallet

V0 includes local approval for demos and internal tools. External channels will be adapters that issue the same proof format, so applications do not need to change policy logic.
