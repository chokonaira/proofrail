# Governance

PermitRail starts as a maintainer-led project.

## Maintainer Responsibilities

Maintainers are responsible for:

- project direction
- release decisions
- security response
- dependency policy
- issue and PR triage
- protecting project marks and attribution

## Decision Principles

The project prioritizes:

1. Security over convenience.
2. Clear policy semantics over hidden magic.
3. Minimal data exposure over broad collection.
4. Provider neutrality over lock-in.
5. Small composable packages over a monolith.

## Path To Shared Governance

When the project has regular external contributors, governance should expand to
include trusted maintainers with clear ownership areas:

- core proof format
- gateway and MCP integration
- provider adapters
- docs and examples
- security response
